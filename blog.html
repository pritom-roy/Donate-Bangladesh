Important FAQ

What is Document Object Model(DOM)?

The Document Object Model (DOM) is like a bridge between web pages and the programming languages used to interact with them, such as JavaScript. When a web page loads, the browser turns it into a structured tree of objects. Each part of the page like - headings, paragraphs, images, becomes a node in this tree. This tree structure is the DOM.

With the DOM, you can use JavaScript to grab elements from the page, change their content, add new ones, or even remove things. It's basically how your code can "talk" to the web page and make it dynamic and interactive, without having to reload the whole page.

How do you select a element from DOM?

To select an element from the DOM, these JavaScript methods are used:

getElementById(): Selects an element by its id.
const myElement = document.getElementById("myId");
getElementsByClassName(): Selects all elements with a specific class.
const elements = document.getElementsByClassName("myClass");
querySelector(): Selects the first element that matches a CSS selector.
const myElement = document.querySelector(".myClass");
querySelectorAll(): Selects all elements that match a CSS selector.
const elements = document.querySelectorAll(".myClass");

These let us pick out stuff from the webpage (DOM) by ID, class, or other selectors like we do with CSS.

What is event delegation in the context of DOM and why is it useful?

Event delegation is a technique in JavaScript that allows handling events more efficiently. Instead of adding an event listener to every single child element in a parent container, only one listener is attached to the parent itself. When an event occurs on one of the child elements, it bubbles up to the parent, which can then decide what to do based on the event's target. This approach leads to fewer event listeners in the code, making it cleaner and often improving performance, especially if there are many child elements.

The cool part about event delegation is that it allows for managing dynamically added elements as well. If a new child element is added later, thereâ€™s no need to attach a new listener; the existing parent listener will catch events from it automatically. This makes it super handy for things like lists, galleries, or any situation where elements can be added or removed frequently. Additionally, it simplifies code and reduces memory usage, which is always a win!

How do you manipulate an element's attributes and style using DOM?

Manipulating an element's attributes and style using the DOM involves a few straightforward methods. To change an element's attributes, the setAttribute method can be used. For example, to change the src of an image, the code would look like this:

const imageElement = document.getElementById('myImage');
imageElement.setAttribute('src', 'newImage.png');

Additionally, the getAttribute method retrieves the current value of an attribute, while the removeAttribute method removes it entirely. For example, to remove a class from an element, the following can be done:

const element = document.querySelector('.myClass');
element.removeAttribute('class');

When it comes to manipulating styles, the style property of an element allows for direct changes to its CSS. For instance, changing the background color and font size can be achieved like this:

const element = document.getElementById('myDiv');
element.style.backgroundColor = 'blue';
element.style.fontSize = '20px';
